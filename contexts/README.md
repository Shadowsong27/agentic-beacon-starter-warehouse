# Contexts

Agent context files for your organization. Each file here is synced to projects
and loaded by AI agents to understand your standards.

## Structure

```
contexts/
└── your-context-name.md   # One file per topic or team standard
```

## What to put here

- Coding standards and conventions
- Deployment and operational procedures
- Code review expectations
- Team-specific patterns and anti-patterns
- Project naming or structure rules

There are no required filenames. Name context files descriptively so agents
and teammates understand what each one contains.

## Example

Create `contexts/python-standards.md` with your Python conventions,
`contexts/git-workflow.md` with your branching and commit rules, and so on.
